Phonegap全記錄How Webs become Apps






一句話說明Phonegap

一言以蔽之，Phonegap就是一種將寫好的Webs封裝成手機Apps的技術。

安裝之前





來到官網， 第一個看到到就是如何insatll，這邊說：To install, ensure that you have NodeJS installed, then open your command-line and run the following. 所以在安裝phonegap之前我們必須要先行安裝NodeJS。接著我們來到NodeJS的下載頁面



筆者這邊使用測試環境的是Macbook pro 2015early 所以在下載畫面出現的是mac osx的版本。隨著不同版本的電腦會出現不同相對應的下載頁面。

接者便可以下載並安裝nodeJS，以下是在筆者電腦的安裝畫面。





在下載資料夾中點選nodeJS的pkg安裝檔(windows大概就是exe檔)，安裝途中就只是一直按確定沒啥特別的。



正式安裝與生成
 在mac中打開terminal(終端機)，或是打開windows中的command line，打入一開始phonegap官網上Install那邊的黑色框框的內文：$ sudo npm install -g phonegap ($號不用打)。這時如果你電腦有密碼，會要求你輸入密碼，輸入完就會看到安裝條在跑。



在安裝之後我們可以在terminal(終端機)中輸入phonegap -h，以觀看phonegap在電腦中適用的指令。其中我們接著會用的便是倒數第三行的create指令。



再來我們就要使用create指令在電腦中創建一個跟phonegap有關的資料夾。我們便在terminal(終端機)中輸入phonegap create test，此時我的一個phonegap資料夾 叫做test便在預設路徑中被下載、創立出來。(圖中間處)



我們可以看到test資料夾已經創建好了。



打開test資料夾其中有個www資料夾，點開後裡面有一個index.html檔。這邊是關鍵所在。



index.html裡面的程式碼原本就只有一個phonegap的icon，而這個index.html就是放你想要轉成App的網頁的地方。以下筆者將 index.html改成一個線上範例的簡易翻頁網頁。網頁內容並非本處重點就不詳加敘述



網頁轉成手機App
 使用phonegap有幾種方法，其一是直接使用終端機並輸入phonegap build生成，其二是使用第三方IDE eclipse來開發。其三是使用Android 開發環境Android Studio來開發。 然而前三種方法多需要複雜的後續安裝及困難環境設定，當一般使用者出現問題時多半不知道要如何解決問題。所以筆者在這邊使用的是最方便和簡單的第四種方法 - 使用phonegap官網上的現成的線上轉換。

我們可以在phonegap官網的首頁中的中段找到一個這樣的地方。他便是線上轉換的入口。



這裡這個功能需要phonegap官網的帳號，所以還請註冊一個帳號。
再來點選try it now按鍵，登入你的帳號(沒有太多用途的話免費帳號就好)，你將看到下列畫面。



圖中你可以看到有兩個選擇，左邊你可以將你自己的github的專案的網址直接貼上去。而右邊你可以上傳你整個phonegap資料夾的壓縮檔(筆者這邊就是test資料夾)。
而筆者本身寫程式就會有git的習慣，所以將選擇左邊方案貼上我自己的 github專案網址。(git相關概念請自行google)
而在這邊有需要注意的， 不管是上傳壓縮檔還是github網址都請移除資料夾中的platforms資料夾。
接著按下Pull from .git repository按鍵就會生成一個可以下載App的頁面了！



而後掃描頁面中的QR Code便可以在手機中下載Android的App安裝檔.APK檔了！
